# JiawenWiki - 个人知识库规划

## 项目概述
基于 Docusaurus 框架构建的个人知识库，用于整理和分享技术知识、学习笔记和个人见解。重点支持 AI 辅助内容创作和多平台内容分发。

## 核心需求
1. **AI 友好的编辑环境**: 使用 VS Code 作为主要编辑器，集成各种 AI 插件辅助写作
2. **多平台内容分发**: 支持一键发布到推特、微信公众号等多个平台
3. **高效的内容创作流程**: 从 Obsidian 知识库迁移，保持高效的写作体验
4. **自动化发布**: 通过脚本和 API 实现内容的自动化分发

## 技术栈
- **框架**: Docusaurus 3.x
- **编辑器**: VS Code + AI 插件 (GitHub Copilot/Claude Dev)
- **部署**: GitHub Pages / Vercel / Netlify
- **版本控制**: Git + GitHub
- **自动化**: GitHub Actions + 自定义脚本
- **多平台 API**: 推特 API、微信公众号 API 等

## 文章创作工作流程

### 📝 标准创作流程
JiawenWiki 采用 AI 辅助的文章创作流程，确保内容质量和效率：

#### 阶段1：主题确定
- **用户提供**：文章主题和大致方向
- **AI 任务**：根据主题生成文章文件框架
- **输出**：带有基础 front matter 和结构的 .md 文件

#### 阶段2：大纲构建  
- **用户提供**：通过语音输入详细说明文章大纲和要点
- **AI 任务**：理解语音内容，整理成结构化大纲
- **输出**：完善的文章大纲和章节结构

#### 阶段3：内容生成
- **用户确认**：大纲结构无误后进行内容填充
- **AI 任务**：基于大纲生成详细内容，保持逻辑性和可读性
- **输出**：完整的初稿文章

#### 阶段4：内容优化
- **用户审阅**：检查内容准确性和表达方式
- **AI 任务**：根据反馈进行修改和优化
- **输出**：最终版本的文章

### 🎯 创作规范

#### 文件命名规范
```bash
# 文件名使用小写字母和连字符
docs/[category]/[topic-name].md

# 示例
docs/build/claude-code-tips.md
docs/wealth/bitcoin-investment-guide.md
docs/health/sleep-optimization.md
```

#### Front Matter 模板
```yaml
---
sidebar_position: [数字]
title: "[文章标题]"
description: "[简短描述，用于SEO和预览]"
tags: [tag1, tag2, tag3, category]
---
```

#### 内容结构标准
1. **引言** - 简要介绍主题和文章价值
2. **核心内容** - 分章节详细阐述
3. **实践应用** - 具体的操作建议或案例
4. **总结** - 要点回顾和行动建议
5. **资源链接** - 相关参考资料（如需要）

### 🗣️ 语音输入处理规范

#### 常见语音转换
- **Cloud → Claude**：自动修正常见的语音识别错误
- **标点符号**：根据语音节奏添加适当的标点
- **专业术语**：确保技术词汇的准确性

#### 大纲整理标准
```markdown
## 大纲示例格式
1. **主要观点1**
   - 支撑要点A

2. **主要观点2**
   - 实例说明
   - 数据支撑
   
3. **实践建议**
   - 具体步骤
   - 注意事项
```
